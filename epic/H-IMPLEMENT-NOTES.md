## 2. Ported functions → REAL source mapping

Every function is a faithful copy of the **production** logic, with the
`ActionHandler::call("name", &json!{...})` JSON indirection collapsed into direct calls.
The only logic removed is the message-loop wrapper (transient `Vec<ChatMessage>`, the 500-token
aggregate gate, the net-savings *batch* guard, observability writes) — which H-ANALYZE assigns
to TS. The **per-message** net-savings guard inside `compress_one` is preserved verbatim.

| Ported fn (in `pluresdb-node/src/headroom.rs`) | Real source | What it does |
|---|---|---|
| `compress_text` (the `compress_one` core) | `headroom_bridge.rs` `compress_one` L231-258 | 200-char floor → route by content type → accept rewrite only if actually smaller & non-empty, else return original. |
| `compress_prose` + `split_sentences` | `headroom_bridge.rs` `compress_prose` L266-301 fused w/ `headroom.rs` `split_sentences` L156-166 | unicode sentence bounds; ≤6 sentences → whitespace-collapse; else keep **first 3 + `[… N sentences elided …]` + last 3**. |
| `compress_code` + `detect_language_heuristic` + `extract_signatures_heuristic` | `headroom_bridge.rs` `compress_code` L310-342 fused w/ `headroom.rs` `detect_language_heuristic` L577-595 + `extract_signatures_heuristic` L600-657 | language sniff → signature-line skeleton w/ `// [headroom: <lang> body elided — N signature(s) kept]` header; no-sigs → whitespace-collapse. |
| `compress_log` | `headroom_bridge.rs` `compress_log` L349-383 | pure-Rust consecutive-duplicate run-collapse → `line  [×N]`; singletons pass through. |
| `collapse_whitespace` | `headroom_bridge.rs` `collapse_whitespace` L406-422 | whitespace-run squeeze → single space, trimmed (json/other + fallback). |
| `detect_content_type` | `headroom.rs` `detect_content_type` actor L68-86 | `{`/`[` → json; `is_log_content` → log; `is_code_content` → code; `looks_like_error` → error; else prose. |
| `count_tokens` | `headroom.rs` `count_tokens` actor L97-103 | real `tiktoken_rs::cl100k_base().encode_with_special_tokens(content).len()`, BPE cached in a process-wide `OnceLock<CoreBPE>` (ported from `headroom.rs` L28-39). |
| `is_log_content` / `looks_like_error` / `is_code_content` | `headroom.rs` L~470-573 | the real structural/indicator heuristics behind `detect_content_type` (timestamp+level density; ≥2 error indicators; brace/indent/semicolon + keyword co-occurrence). |

**NAPI surface added** (`lib.rs`, `#[napi]` free functions):
```ts
compressText(content: string, contentType?: string | null): string   // compress_one router + 4 strategies
countTokens(text: string): number                                     // real cl100k_base
detectContentType(content: string): string                           // 'json'|'log'|'code'|'error'|'prose'
```
`contentType` is an optional second arg (not an object) — the cleaner NAPI mapping; when
provided it pins the route the TS seam already knows, otherwise the real detector classifies.

   out (they belong to the later TS seam).

algorithm was **copied** (it is pure and self-contained), not imported across the boundary.
